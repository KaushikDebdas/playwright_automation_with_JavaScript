How to use this site

Save this HTML file locally and serve it (or just open it in your browser). Point Playwright at file:// path or run a local server:

# Quickest: just use the file path
await page.goto('file:///path/to/playwright-practice-site.html');

# Or with python's built-in server (recommended for API tests)
$ python3 -m http.server 8080
await page.goto('http://localhost:8080/playwright-practice-site.html');

Every interactive element has a data-testid attribute, plus meaningful id, name, and ARIA labels — practice every selector strategy.

Advanced Dropdowns

          Real-world patterns
        

        
Native <select> is rare in modern admin panels. These are the dropdowns you'll actually meet — custom div-based widgets, async-loaded options, dynamic IDs that change every render, cascading dependencies, and per-row dropdowns. Each card explains the pain point and the right Playwright approach.


        
        

          
AD-01Custom Div-based Dropdown · Dynamic IDs

          

            

              The pain: This isn't a native <select> — it's built with divs (like React Select / MUI). Standard selectOption() won't work. Worse, every option has a freshly generated UUID as its id attribute on every render — selectors using those IDs will fail on the next page load. Click "Re-render" below and inspect the DOM to see IDs change.


              The right approach: Scope to the dropdown via stable data-testid, then locate options by their visible text or data-value attribute — never by the dynamic id.

AD-04Async-loaded Dropdown

          

            

              The pain: Options aren't in the DOM until you open the dropdown — they're fetched on first open with a 1.5s delay. Asserting on options before they load gives flaky tests.


              Practice: Open dropdown, wait for the loading state to disappear, then interact. Use expect(locator).toBeVisible() with a timeout, or wait for the loader to detach.

          05 ──
          
API Testing

          REST
        

        
Playwright includes request — a built-in HTTP client perfect for API tests, auth setup, and hybrid UI+API workflows. Use the public APIs below to practice without setting up a backend.


        
        

          
A-01Recommended Public APIs

          

            
All APIs below are free, require no auth (mostly), and accept GET/POST/PUT/DELETE. Perfect for learning.

            

              JSONPlaceholder — fake REST API for testing

              https://jsonplaceholder.typicode.com
              
Simulates real responses but doesn't actually persist data. Great for status code & payload assertions.

            

            

              ReqRes — hosted REST mock

              https://reqres.in/api
              
Includes login/register endpoints with token responses — ideal for auth flow tests.

            

            

              DummyJSON — faker-style data, real persistence simulation

              https://dummyjson.com
              
Auth, products, carts, posts, comments — most comprehensive for end-to-end practice.

            

            

              Fake Store API — e-commerce mock

              https://fakestoreapi.com
              
Products, carts, users — practice e-commerce test scenarios.

            

            

              Restful-Booker — hotel booking with real auth

              https://restful-booker.herokuapp.com
              
Token-based auth, full CRUD on bookings. Excellent for chained API tests.

            

          

        


        
        

          
A-02Endpoint Reference (use any of the APIs above)

          

            
Copy these patterns to request.get/post/put/delete calls.

            

              GET/posts
              
List all posts. Assert status() === 200, response is array, length > 0.

            

            

              GET/posts/1
              
Single resource. Assert id === 1, body has title, userId.

            

            

              POST/posts
              
Create resource. Send JSON body, assert 201, response includes the new id.

            

            

              PUT/posts/1
              
Replace resource. Assert updated fields are returned.

            

            

              PATCH/posts/1
              
Partial update. Only the changed field should differ.

            

            

              DELETE/posts/1
              
Remove resource. Assert 200 or 204, then GET should 404.

            

          

        


        
        

          
A-03Playwright API Test Examples

          

            
Drop these into your .spec.ts files to start practicing immediately.


            
1. Simple GET assertion

            
import { test, expect } from '@playwright/test';

test('GET /posts/1 returns expected post', async ({ request }) => {
  const res = await request.get('https://jsonplaceholder.typicode.com/posts/1');
  expect(res.status()).toBe(200);

  const body = await res.json();
  expect(body).toHaveProperty('id', 1);
  expect(body.title).toBeTruthy();
});


            
2. POST with body

            
test('POST /posts creates a resource', async ({ request }) => {
  const res = await request.post('https://jsonplaceholder.typicode.com/posts', {
    data: { title: 'My Test', body: 'Hello', userId: 1 }
  });
  expect(res.status()).toBe(201);
  const body = await res.json();
  expect(body.title).toBe('My Test');
});


            
3. Auth flow with token capture

            
test('login then use token', async ({ request }) => {
  // Step 1: login
  const login = await request.post('https://dummyjson.com/auth/login', {
    data: { username: 'emilys', password: 'emilyspass' }
  });
  expect(login.ok()).toBeTruthy();
  const { accessToken } = await login.json();

  // Step 2: use token
  const me = await request.get('https://dummyjson.com/auth/me', {
    headers: { Authorization: `Bearer ${accessToken}` }
  });
  expect(me.status()).toBe(200);
});


            
4. Hybrid UI + API (login via API, then test UI)

            
test('API login + UI assertion', async ({ page, request }) => {
  // Get token via API (faster than UI login)
  const res = await request.post('https://dummyjson.com/auth/login', {
    data: { username: 'emilys', password: 'emilyspass' }
  });
  const { accessToken } = await res.json();

  // Inject into localStorage before page loads
  await page.addInitScript(token => {
    localStorage.setItem('token', token);
  }, accessToken);

  await page.goto('/dashboard');
  await expect(page.getByTestId('welcome-msg')).toBeVisible();
});


            
5. Mocking API responses (network interception)

            
test('mock API response', async ({ page }) => {
  await page.route('**/api/users', route => {
    route.fulfill({
      status: 200,
      body: JSON.stringify([{ id: 1, name: 'Mocked User' }])
    });
  });

  await page.goto('/users');
  await expect(page.getByText('Mocked User')).toBeVisible();
});
